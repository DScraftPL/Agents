# Objectives
- Implement a login screen as the initial entry point for the website.
- Restrict access to the map and data, making them visible only to authenticated users.
- Ensure user credentials are validated before granting access to the main website features.
- Provide a secure authentication process to protect user access.

### [markdown] TASK.md > Problem Statement
# Problem Statement
The website currently allows users to access the map and data without authentication. It requires a login screen to ensure that only authorized users can view the map and associated data.

    <script>
        document.getElementById('form').addEventListener('submit', function(event) {
            event.preventDefault();
            const username = document.getElementById('username').value;
            const password = document.getElementById('password').value;

            const headers = new Headers();
            headers.set('Authorization', 'Basic ' + btoa(username + ':' + password));

            fetch('http://localhost:5000/api/penguins', { headers: headers })
                .then(response => {
                    if (response.ok) {
                        window.location.href = 'index.html';
                    } else {
                        alert('Invalid credentials');
                    }
                });
        });
    </script>
</body>
